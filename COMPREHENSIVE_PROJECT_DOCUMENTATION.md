# 基于深度学习的实时交通流预测系统 - 综合项目文档

## 项目概述
本项目实现了基于ASTGCN（Attention Based Spatial-Temporal Graph Convolutional Networks）的实时车辆速度预测系统，包含前后端分离的可视化架构。系统旨在通过深度学习技术对城市交通流进行精准预测，为智慧交通管理提供数据支撑。

## 解决的关键技术问题

### 1. 原始内存溢出问题
- **问题**: `numpy.core._exceptions.MemoryError: Unable to allocate 1.26 MiB for an array with shape (108, 307, 5)`
- **解决方案**: 实现了分批处理和预检查机制，避免一次性加载所有数据到内存

### 2. 模型维度不匹配问题
- **问题**: `mat1 and mat2 shapes cannot be multiplied (307x1 and 5x64)` 等维度错误
- **根本原因**: 
  - 输入数据的特征维度与模型期望不符
  - 数据形状 `(nodes, features, timesteps)` 与模型期望 `(batch, nodes, features, timesteps)` 不匹配
- **解决方案**:
  - 修正特征数从5到3（根据PEMS04数据集实际特征数）
  - 正确重构输入数据形状为 `(batch, nodes, features, timesteps)`
  - 确保数据预处理流程与模型架构匹配

### 3. 模型参数缺失问题
- **问题**: `UpgradeASTGCN.__init__() missing 1 required positional argument: 'num_of_features'`
- **解决方案**: 修正模型初始化参数

### 4. 设备兼容性问题
- **问题**: `RuntimeError: Expected all tensors to be on the same device`
- **解决方案**: 确保数据和模型在相同设备上处理

## 系统架构

### 后端：高并发流式推理引擎
- **WebSocket双向通信**: 实现低延迟实时数据传输
- **滑动窗口机制**: 维护固定长度的双端队列（长度36，代表过去3小时的36个5分钟时间步）
- **异步推理引擎**: 使用线程池避免阻塞数据流
- **数据生产者**: 从PEMS04真实数据集读取数据，实现真实时间序列预测

### 前端：数据可视化大屏
- **中央地图**: Leaflet展示307个传感器节点
- **右侧预测面板**: ECharts展示"历史实线+未来虚线"标准预测视图
- **左侧监控面板**: 拥堵指数、性能指标和注意力热力图
- **暗色科技风**: 现代化数据大屏设计

## 核心功能模块

### 1. 后端：高并发流式推理引擎
- **双向长连接 (WebSocket)**: 摒弃传统的HTTP Request-Response模式，采用WebSocket实现双向实时通信
- **数据缓冲区 (Sliding Window)**: 在内存中维护固定长度的双端队列，长度为12（代表过去3小时的36个5分钟时间步）
- **数据生产者**: 异步后台任务，从原始数据中按时间轴逐帧读取真实数据，时间缩放机制将现实中的5分钟压缩为1秒钟
- **推理引擎**: 每次缓冲区更新后触发模型推理，使用线程池避免阻塞数据流
- **实时推送**: 将当前时刻的真实路网状态+未来3小时的预测路网状态打包为JSON，通过WebSocket推送给所有客户端

### 2. 前端：暗色系科技风数据大屏
- **中央主视觉 - 时空路网动态演变图**: 使用Leaflet展示307个传感器节点，气泡大小代表车流量，颜色代表平均车速
- **右侧面板 - 微观断面预测与对比**: 使用ECharts展示真实车速（实线）和预测车速（虚线）的对比
- **左侧面板 - 全局监控与模型可解释性**: 显示拥堵指数、MAE/RMSE指标和自适应图注意力矩阵热力图

### 3. 右侧面板 - "历史实线 + 未来虚线"标准视图
- 实现了学术界标准的预测视图
- X轴时间窗口：过去3小时到未来3小时（共72个5分钟时间步）
- 加入"当前时刻"分割线（NOW标记的垂直虚线）
- 历史数据：左侧实线表示真实历史车速
- 预测数据：右侧虚线表示模型预测车速，带半透明置信区间阴影

### 4. 右侧面板 - "未来1小时最拥堵节点TOP 5预警"
- 新增柱状图显示未来1小时最拥堵的5个节点
- 按拥堵指数降序排列，便于交通管理部门优先关注
- 渐变色彩编码，直观反映拥堵程度

### 5. 左侧面板 - 注意力机制交互增强
- 在注意力热力图下方添加说明文字："高亮色块代表两相隔甚远的路口存在强烈的交通流传导关联"
- 实现节点选择联动：当选中地图上的某个节点时，热力图高亮显示该节点相关的行
- 突出显示选中节点与其他节点的关联关系

### 6. 数据流处理
- **真实数据源**: 直接从PEMS04数据集读取真实交通数据 (16992, 307, 3)
- **连续时间流**: 按时间顺序依次处理数据点
- **滑动窗口**: 维护36个时间步的滑动窗口用于预测

### 7. 预测算法
- **ASTGCN模型**: 使用升级版时空图卷积网络
- **时空注意力**: 捕获空间和时间依赖关系
- **多尺度预测**: 基于week/day/recent三个时间尺度

### 8. 可视化展示
- **实时地图**: 显示307个传感器节点的实时状态
- **预测对比**: 展示真实值vs预测值的对比
- **拥堵监测**: 实时计算和展示拥堵指数
- **注意力热力图**: 可视化节点间关联关系
